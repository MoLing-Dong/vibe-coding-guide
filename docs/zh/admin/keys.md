# 需求与验收

Vibe Coding 最容易失败的地方，不是模型不够聪明，而是需求没有被说清楚。把需求和验收标准写好，AI 才能稳定地交付你真正想要的结果。

## 需求卡片

一张好用的需求卡片包含：

- 背景：为什么要做。
- 用户：谁会使用。
- 行为：用户会怎么操作。
- 规则：有哪些业务限制。
- 边界：这次不做什么。
- 验收：怎样算完成。

## 验收标准写法

不要只写“优化体验”。改成可检查的句子：

| 模糊说法 | 更好的写法 |
| --- | --- |
| 页面更好看 | 表单按钮在 375px 宽度下不换行遮挡 |
| 修复筛选问题 | 选择“已退款”后 URL 出现 `refundStatus=refunded` |
| 提升性能 | 首屏接口请求数量不超过 2 个 |
| 增加错误提示 | 保存失败时显示服务端返回的错误文案 |

## 交给 AI 前的检查

- 是否说明本次任务不包含哪些内容。
- 是否有至少一个可验证结果。
- 是否包含异常状态和空状态。
- 是否说明旧数据、权限或兼容性要求。
- 是否写明要保留的现有行为。

需求越具体，AI 越像合作者；需求越模糊，AI 越像猜谜机器。
