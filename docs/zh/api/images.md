# 视觉资料

截图、设计稿、白板照片和报错截图都能显著提升 Vibe Coding 效率。关键是不要只丢图，还要说明你希望 AI 从图里判断什么。

## 适合提供的视觉资料

- 页面截图：指出布局、间距、状态、交互问题。
- 设计稿：说明哪些是必须还原，哪些可以按现有组件调整。
- 报错截图：同时提供文字错误栈，避免图片识别遗漏。
- 白板草图：说明业务流程和模块关系。
- 对比图：标记“当前效果”和“期望效果”。

## 提供方式

```txt
这张图是订单详情页当前效果。

请重点看：
1. 顶部状态栏和按钮是否拥挤。
2. 退款信息是否应该放在右侧摘要区。
3. 移动端宽度下哪些元素需要换行。

约束：
- 复用现有 Button、Tag、DescriptionList 组件。
- 不新增全局样式。
先给布局调整建议，不要直接改代码。
```

## 从图片到代码的流程

1. 让 AI 描述它看到的关键元素。
2. 纠正识别错误。
3. 补充现有组件和代码路径。
4. 要求它先给实现方案。
5. 确认方案后再改代码。
6. 用截图或浏览器检查最终效果。

## 注意事项

- 图片不能替代验收标准。
- 对尺寸、颜色、文案有硬要求时要写成文字。
- 模糊、压缩严重或遮挡的截图容易误导 AI。
- 涉及隐私信息时先打码。
- 视觉还原任务要明确桌面端和移动端都要检查。
