# 控制成本与上下文

Vibe Coding 的成本不只来自模型费用，也来自时间、注意力和上下文污染。越早把任务边界讲清楚，越少出现“聊了很久但代码不可用”的情况。

## 常见概念

| 概念 | 说明 |
| --- | --- |
| 上下文预算 | 当前任务真正需要 AI 理解的信息量 |
| 对话漂移 | 对话过长后，AI 忘记目标或混入旧约束 |
| 验证成本 | 运行测试、构建、手动检查所需时间 |
| 返工成本 | 因需求不清或范围过大导致的重复修改 |
| 切换成本 | 在多个助手、分支或任务之间来回切换 |

## 控制上下文

1. 只给和任务相关的文件、错误、截图和需求。
2. 长任务先让 AI 写一份当前理解，再继续执行。
3. 当对话开始混乱时，新开会话并粘贴简短交接摘要。
4. 不要把多个互不相关的需求塞进同一轮。
5. 让 AI 每次改动前复述目标和约束。

## 控制时间

- 15 分钟还没找到相关文件，就先停下来重新描述问题。
- 30 分钟没有可运行结果，就缩小任务范围。
- 每完成一个小步骤就验证，不把验证留到最后。
- 对重复失败的问题，要求 AI 先列假设和证据。

## 控制模型花费

- 小任务用便宜快速的助手，复杂推理再切到强模型。
- 大段日志先裁剪，只保留错误栈、复现步骤和相关配置。
- 让 AI 读文件前说明要找什么，避免无目标浏览。
- 让 AI 生成补丁而不是反复输出完整文件。

## 停止信号

出现下面情况时，暂停当前会话：

- AI 开始修改无关文件。
- 它无法解释自己改动的原因。
- 同一个测试反复失败但没有新假设。
- 需求已经变化，但旧对话还在影响输出。
